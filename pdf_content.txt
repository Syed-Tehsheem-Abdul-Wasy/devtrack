UV/UVX, FastAPI, SQLModel, and tooling
Why UV/UVX?
UV is a next-generation Python package manager written in Rust. It replaces pip, pip-tools, virtualenv, and pyenv
in a single blazing-fast binary — 10–100× faster than pip. In 2025, UV/UVX is the industry default for professional
Python projects.
Project Initialization
# Install UV (one-time)
curl -LsSf https://astral.sh/uv/install.sh | sh
# Create project
uv init devtrack
cd devtrack
# Add runtime dependencies
uv add fastapi[standard] sqlmodel psycopg2-binary
uv add python-jose[cryptography] passlib[bcrypt] python-dotenv
# Add dev dependencies
uv add --dev pytest pytest-asyncio httpx ruff
# Run the development server
uv run fastapi dev app/main.py

Key Concepts & Code Patterns
Study these before you start coding
SQLModel — Unified Model + Schema
SQLModel lets you define one class that serves as both the SQLAlchemy table and a Pydantic validation
schema. Use table=True for DB models and create separate Input/Output schemas by inheriting from the base
class.
from sqlmodel import SQLModel, Field
from typing import Optional
from datetime import datetime
class TaskBase(SQLModel):
    title: str = Field(min_length=1, max_length=200)
    description: Optional[str] = None
    priority: str = Field(default="medium")  # low | medium | high
    status: str = Field(default="todo")      # todo | in_progress | done
class Task(TaskBase, table=True):            # ← DB table
    id: Optional[int] = Field(default=None, primary_key=True)
    project_id: int = Field(foreign_key="project.id")
    assignee_id: Optional[int] = Field(default=None, foreign_key="user.id")
    created_at: datetime = Field(default_factory=datetime.utcnow)
class TaskCreate(TaskBase): pass             # ← POST body schema
class TaskRead(TaskBase):                    # ← Response schema
    id: int
    project_id: int
    created_at: datetime

from jose import jwt, JWTError
from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/login")
def create_access_token(data: dict) -> str:
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(minutes=settings.token_expire)
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, settings.secret_key, algorithm=settings.algorithm)
def get_current_user(
    token: str = Depends(oauth2_scheme),
    session: Session = Depends(get_session)
) -> User:
    exc = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Invalid or expired token",
        headers={"WWW-Authenticate": "Bearer"}
    )
    try:
        payload = jwt.decode(token, settings.secret_key,
                             algorithms=[settings.algorithm])
        user_id = int(payload.get("sub"))
    except (JWTError, TypeError):
        raise exc
    user = session.get(User, user_id)
    if not user: raise exc
    return user
FastAPI Router Pattern
from fastapi import APIRouter, Depends, HTTPException, status
from sqlmodel import Session, select
router = APIRouter(prefix="/projects", tags=["Projects"])
@router.post("/", response_model=ProjectRead, status_code=201)
def create_project(
    project_in: ProjectCreate,
    session: Session = Depends(get_session),
    current_user: User = Depends(get_current_user)
):
    project = Project(**project_in.model_dump(), owner_id=current_user.id)
    session.add(project)
    session.commit()
    session.refresh(project)   # ← always refresh after commit!
    return project
# Register in main.py:
app.include_router(projects_router)

■  Tip: Always call session.refresh()
After session.commit(), call session.refresh(obj) to reload server-generated values (id,

Academic Integrity & Pro Tips
How to succeed and what not to do
■■  Academic Integrity Policy
You may reference FastAPI docs, SQLModel docs, and online tutorials. You may NOT copy another
student's code or submit AI-generated code you don't understand. If asked, you must explain any line
of your code. Violations result in a zero for the entire capstone.
Pro Tips From the Industry
 Read the FastAPI docs first. — The official tutorial at fastapi.tiangolo.com covers 80% of what you need.
Spend 2 hours before writing a single line.
 Commit early, commit often. — Push to GitHub after every phase. If something breaks, you can roll back.
Employers look at commit history.
 Test in Swagger UI as you build. — FastAPI auto-generates /docs. Use it after every endpoint. Don't wait
until the end to discover broken routes.
 Never hard-code credentials. — All secrets go in .env. Use Pydantic BaseSettings to load them. This is
non-negotiable in real projects.
 Return proper HTTP status codes. — 404 for not-found, 403 for forbidden, 400 for bad input. A 200
response for a missing resource is a fail.
 Write your README for a stranger. — Can someone with no context clone and run it in 5 minutes? If not,
improve the README.
 Deploy by end of Phase 3. — Railway deployment takes time to debug. Don't leave it to the last hour before
submission.
